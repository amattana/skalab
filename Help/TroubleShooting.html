DAQ Error


The DAQ requires some special config to work, in particular the Python should run with sudo privileges.


Run the following command to let the python to use the DAQ correctly by any user:


sudo setcap cap_net_raw,cap_ipc_lock,cap_sys_nice,cap_sys_admin,cap_kill+ep /usr/bin/python3.10


depending of which Python version you are using





QT5 / GNOME / Wayland


Some QT5 library methods may not work properly or do not work at all with Ubuntu 22

Wayland is a communication protocol that specifies the communication between a display server and its clients.

By default the Ubuntu 22.04 Jammy Jellyfish desktop already uses Wayland but itâ€™s also possible to load to Xorg display server instead.


To disable Wayland you can Edit the following file:


$ sudo nano /etc/gdm3/custom.conf


and set


WaylandEnable=false


Restart the service with:


$ sudo systemctl restart gdm3

or just reboot.
